## Что это за проект

**Vibe Coding Playground** — одностраничный интерактивный showcase-сайт, демонстрирующий возможности современной веб-разработки. Каждый блок страницы — отдельная "витрина" с живыми примерами: анимации, интерактив, визуальные эффекты, адаптивность.

Проект создаётся как **пример того, что можно сделать за 1 день с помощью Claude + Cursor** — это и демонстрация мощи AI-ассистированной разработки, и учебный материал.

---

## Цели проекта

| Цель | Описание |
|------|----------|
| **Вдохновить** | Показать новичкам и опытным разработчикам, что современный веб — это не скучные страницы |
| **Обучить** | Стать live-примером для учеников курса по вайбкодингу |
| **Впечатлить** | Каждый блок должен вызывать реакцию «о, как это сделано?» |
| **Продвинуть** | Footer ведёт к курсу / GitHub — мягкий CTA |

---

## Целевая аудитория

- **Новички** — видят, что можно создать, и хотят научиться
- **Ученики курса** — используют как справочник и референс
- **Опытные разработчики** — вдохновляются техниками и паттернами
- **Рекрутеры / клиенты** — видят уровень возможностей

## Структура страницы (секции)

```
[Hero]          → Wow-эффект при загрузке, заголовок, подзаголовок
[Animations]    → Stagger, scroll-reveal, hover-анимации
[Interactivity] → Кнопки с конфетти, слайдеры, морфинг
[Visual Effects]→ Glassmorphism, glow, градиенты, neon
[Responsive]    → Переключатель device-режимов, сетки
[Footer]        → GitHub, CTA, credits
```

---

## Принципы дизайна

1. **Каждый блок — отдельная история** с собственным визуальным языком
2. **Тёмная тема по умолчанию** + переключатель в светлую
3. **Анимации ненавязчивы** — не мешают восприятию контента
4. **Всё интерактивно** — пользователь должен хотеть кликать и двигать
5. **Mobile-first** — адаптивность не опциональна
6. **Performance** — 90+ Lighthouse, анимации через GPU (transform/opacity)
7. **Accessibility** — `prefers-reduced-motion`, семантика, ARIA

---

## Что НЕ делаем

- Не делаем CMS или бэкенд — это статичный сайт
- Не используем реальные данные — всё демо-контент
- Не добавляем аналитику на MVP
- Не делаем мультиязычность на старте

## Ключевые решения

| Вопрос | Решение | Причина |
|--------|---------|---------|
| Фреймворк | Next.js 14 App Router | SSG, оптимизация, экосистема |
| Анимации | Framer Motion | Декларативно, мощно, React-нативно |
| Стили | Tailwind CSS v4 | Скорость разработки, утилитарность |
| 3D/канвас | Three.js (Hero только) | WOW-эффект без перегруза |
| Темы | next-themes | Простая реализация dark/light |
| Деплой | Vercel | Нулевая конфигурация для Next.js |
